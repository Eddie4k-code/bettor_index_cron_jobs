# Ingestion and Hit Rate Calculation Pipelines

## Purpose
This project contains pipelines for ingesting player prop data for various sports leagues and calculating hit rates for those props based on player statistics.

## Data Source For Gambling Data, Odds Data and Player Prop Data
- **balldontlie API**: Used for fetching player prop bets, teams, players, player stats, and player games.
  - API Reference: https://balldontlie.io/openapi.yml

## Coding Conventions
- Always use the balldontlie API for sport related player, team, and stats data.
- When ingesting player props, ensure all relevant player, team, and game context is fetched and stored.
- Hit rate calculations must be based on actual player stats from the ingested data.
- Document any new pipeline with a docstring at the top describing its purpose, data flow, and dependencies.
- Prefer clear, modular functions for each step: ingestion, transformation, calculation, and storage.
- Use environment variables for API keys and sensitive configuration (see `.env`).
